상속
      일반적으로 어떤 개념을 계층화하여 분류하면 전체 그림이 그려져 더 쉽게 이해를 할 수 있다.
      소프트웨어 개발자들도 클래스 간의 관계를 계층화하고 분류하는데 이런 개념이 바로 상속(inheritance)이다.

      일상에서 상속은 '부모에게서 재산을 상속받는다'는 형태로 많이 사용된다. 즉 뭔가를 물려받는다는 의미이다. 이처럼 객체지향에서도 상위 클래스(super class)의 모든 것을 하위 클래스(sub class)가 물려받아 내 것처럼 사용함을 의미한다.
      이때 물려주는 클래스를 상위 클래스 또는 부모 클래스라고 하고, 물려받는 클래스를 하위 클래스 또는 자식 클래스라고 한다.

      예를 들어, [그림 6-33]에서 상위 클래스 A는 데이터 3개(a, b, c)와 메서드 3개(funcA(), funcB(), funcC())를 가지고 있다. 또 하위 클래스 B는 데이터 2개(d, e)와 메서드 2개(funcD(), funcE())를 가지고 있다. 그리고 다른 하위 클래스 C도 데이터 2개(f, g)와 메서드 2개(funcF(), funcG())를 가지고 있다.
      여기서 상속이라는 개념을 사용하면 하위 클래스 B와 C가 상위 클래스 A에 선언된 데이터 3개와 메서드 3개를 마치 자기 것인양 마음대로 쓸 수 있다는 것이다.

      따라서 하위 클래스 B는 데이터 5개(a, b, c, d, e)와 메서드 5개(funcA(), funcB(), funcC(), funcD(), funcE())를 가지고 있는 것과 같다.
      클래스 C에도 마찬가지로 적용된다.
